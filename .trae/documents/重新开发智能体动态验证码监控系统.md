# 重新开发智能体动态验证码监控系统计划

## 1. 任务概述

根据提供的开发文档，删除当前目录下的所有文件，重新开发智能体动态验证码监控系统。

## 2. 技术栈

* **后端**：PHP 7.4+，MySQL 5.7+，PDO，JWT，RESTful API

* **前端**：HTML5/CSS3，JavaScript，Bootstrap，Fetch API

## 3. 开发步骤

### 3.1 删除现有文件

* 删除当前目录下的所有文件和子目录

### 3.2 创建目录结构

按照文档中定义的目录结构创建文件夹：

```
├── js/                    # 前端模块化代码
├── api/                    # 后端API目录
│   └── utils/              # 工具类
└── public/                 # 公共资源
```

### 3.3 开发后端代码

1. **工具类开发**：

   * `api/utils/Database.php` - 数据库连接类

   * `api/utils/JwtUtils.php` - JWT工具类

   * `api/utils/Response.php` - 响应处理类

   * `api/utils/Log.php` - 日志工具类

2. **API开发**：

   * `api/config.php` - 系统配置文件

   * `api/auth.php` - 认证API

   * `api/monitors.php` - 监控管理API

   * `api/codes.php` - 验证码API

   * `api/admin.php` - 管理员API

   * `api/api_settings.php` - API设置API

3. **数据库脚本**：

   * `database.sql` - 数据库表结构脚本

### 3.4 开发前端代码

1. **HTML页面**：

   * `index.html` - 首页

   * `login.html` - 登录页面

   * `register.html` - 注册页面

   * `dashboard.html` - 仪表盘

   * `monitors.html` - 监控管理

   * `codes.html` - 验证码历史

   * `api-settings.html` - API设置

   * `settings.html` - 用户设置

   * `admin.html` - 管理员页面

2. **JavaScript模块化代码**：

   * `js/api.js` - API通信模块

   * `js/app.js` - 应用主入口

   * `js/auth.js` - 认证模块

   * `js/code.js` - 验证码模块

   * `js/monitor.js` - 监控器模块

   * `js/utils.js` - 工具模块

## 4. 开发规范

* 遵循PSR-4自动加载规范

* 代码缩进使用4个空格

* 变量命名使用驼峰命名法

* 函数命名使用小写字母加下划线

* 表名使用小写字母加下划线

* 所有用户输入必须经过验证

* 密码必须使用bcrypt加密

* 敏感数据必须加密存储
