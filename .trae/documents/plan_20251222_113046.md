# 修复API设置部分的错误并添加API URL功能

## 问题分析

根据用户反馈和截图，API设置部分存在以下问题：

1. **标签文字重复**："API密钥密钥" 标签重复，应该修正
2. **缺少API URL功能**：需要添加根据当前网址生成的API URL，用于获取监控列表下的所有短信全文

## 解决方案

1. **修复标签文字**：将重复的标签修正为合理的文字
2. **添加API URL字段**：生成基于当前域名的API URL，包含用户的API密钥
3. **增强用户体验**：添加复制按钮，方便用户复制API URL

## 实施步骤

1. **修改HTML结构**：
   - 修正重复的标签文字
   - 添加API URL字段和复制按钮

2. **更新JavaScript功能**：
   - 添加生成API URL的函数
   - 添加复制API URL到剪贴板的功能
   - 监听API密钥变化，自动更新API URL

3. **调整样式**：
   - 确保新增字段与现有布局一致
   - 优化用户体验

## 预期结果

1. **修正后的标签**：不再有重复文字，标签清晰准确
2. **新增API URL**：根据当前域名和用户API密钥生成完整的API URL
3. **复制功能**：用户可以一键复制API URL
4. **动态更新**：当API密钥变化时，API URL自动更新

## 技术实现

- 使用JavaScript的`window.location.origin`获取当前域名
- 使用`navigator.clipboard.writeText`实现复制功能
- 监听API密钥输入框的变化事件
