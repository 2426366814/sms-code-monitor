# 修复管理员API资源路径解析问题

## 问题分析
从代码检查来看，获取用户列表失败的原因是管理员API无法正确解析资源路径。当请求路径是`api/admin/users`时：
1. `backend/index.php` 正确路由到 `backend/api/admin/index.php`
2. 但在 `admin/index.php` 中，它依赖于 `$_SERVER['PATH_INFO']` 来获取资源路径
3. 而 `PATH_INFO` 可能不存在，导致 `$resource` 为空字符串
4. 因此无法进入 `case 'users'` 分支处理用户列表请求

## 解决方案
修改 `backend/api/admin/index.php` 文件，在 `PATH_INFO` 不存在时，使用 `REQUEST_URI` 来解析资源路径，确保能正确获取到 `users` 资源。

## 修复步骤
1. 修改 `backend/api/admin/index.php`：在获取资源路径时，增加对 `REQUEST_URI` 的解析
2. 测试获取用户列表功能，确保问题已经解决
3. 重新打包项目

## 预期结果
修复后，管理员API能够正确解析资源路径，无论 `PATH_INFO` 是否存在，都能正确处理 `api/admin/users` 请求，获取用户列表功能能够正常工作。
